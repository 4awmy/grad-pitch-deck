Slide 1: The Problem
The Knowledge Barrier & Retrieval Bottleneck
The Current State
Students and staff depend on scattered PDFs, portal pages, email threads, and office visits. Answers are slow, inconsistent, and hard to verify.
Technical Friction
General-purpose chat tools hallucinate, lose context, and cannot cite university documents with enough precision for policy, admission, or course questions.
The Core Dilemma
Users are forced to choose between manual searching and external tools that are not grounded in the university's own sources.
The Bottleneck: Generic LLM vs. Grounded Retrieval
Without retrieval and citations, a model can answer quickly but incorrectly. A university assistant needs source control, confidence thresholds, and traceable responses.
👨🏫
Presenter Note for the Doctor
"We are targeting the gap between useful generative answers and verifiable academic answers. The real problem is not generation alone, but trusted grounding in local university knowledge."
Slide 2: What We Build
Local LLM + RAG Assistant for the University

A self-contained assistant that answers academic and administrative questions from university documents, policies, and course materials without relying on ungrounded public responses.

1. Document Grounding
Indexes university PDFs, guides, and announcements so the assistant answers from approved material instead of from memory.
2. Retrieval + Generation
Uses vector search to fetch relevant passages, then generates concise answers that stay close to the retrieved text.
3. Usable Output
Returns answers with citations, source snippets, and a clear path for users to verify the result.
👨🏫
Presenter Note for the Doctor
"This is not a generic chatbot. It is a retrieval-backed university assistant with a controlled document set, so every answer can be traced back to institutional sources."

Slide 4: Research Angle
Academic Contribution & Evaluation

The Thesis Question: How much does retrieval-grounding improve answer reliability, citation quality, and student usability compared with a plain LLM on university questions?

Execution Speed
Measure retrieval latency, answer generation time, and end-to-end response speed for common student queries.
Accuracy & Faithfulness
Evaluate whether the answer matches the source passages and whether it avoids unsupported claims.
Source Usefulness
Check whether students can actually use the citations to verify policy and course answers without extra searching.
Scalability
Test how performance changes as the corpus grows from a few documents to a larger university knowledge base.
👨🏫
Presenter Note for the Doctor
"The research value is in controlled evaluation: answer correctness, source grounding, and the practical value of citations for a real university environment."
