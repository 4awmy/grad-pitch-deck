The Problem

Local Edge-AI Deployment Bottlenecks for Small Enterprises

The Current State

Small businesses still rely on manual data processing or expensive cloud subscriptions to handle invoicing, customer queries, and document retrieval.

The Technical Friction

Open-source tools fail because they create setup complexity and hardware mismatch on consumer-grade machines with limited RAM and compute.

The Core Dilemma

Users are forced to choose between vulnerable cloud APIs and local AI stacks that require deep Linux and Docker expertise to orchestrate.

Slide 2

What We Build

Edge-AI Provisioner: An Automated Constraint-Aware Engine

The Core System: A self-contained local application designed to automate and optimize the architectural design and deployment of local AI environments based on hardware constraints.

1. Automated Hardware Auditing

Parses host specs and available VRAM/RAM without manual user input.

2. Constraint-Aware Routing

Matches business needs to hardware-appropriate open-weight models through a constraint satisfaction algorithm.

3. Actionable Execution

Delivers a ready-to-deploy Docker Compose package or installation script tailored to the target machine.

How It Works

Input & Preprocessing Layer: Raw system constraints and business logic inputs are normalized into a structured JSON profile.

Core Processing Engine: A deterministic constraint satisfaction engine matches models and vector DBs to available memory.

Output & Integration Layer: Processed results are exposed alongside customized, ready-to-execute provisioning scripts.

Scoping & Implementation Tiers

Project Feasibility & Scalable Execution Tiers

Tier 1 - The MVP

(Guaranteed Graduation Deliverable)

Scope: A local web app that profiles hardware, interviews the user, and runs the constraint algorithm against a static catalog of five key archetypes.

Tier 2 - Target System

(Full Academic Scope)

Scope: End-to-end integration where the engine auto-spins Docker containers locally and includes a complete benchmarking suite.

Tier 3 - Future Work

(Out of Scope / Stretch Goals)

Scope: Distributed AI clusters across office PCs, auto-installing Linux on bare metal, or autonomous self-learning feedback loops.
